# Guide Main Home

本项目是一个用于归档和发布多个独立 HTML 页面的仓库。其目的在于集中存放零散的、不隶属于特定网站或项目的静态页面资源，方便统一管理、版本控制以及通过 GitHub Pages 或其他静态托管服务进行访问。

## 项目简介

本仓库不绑定任何特定域名，也不指向某个具体网站。所有内容均为独立的 HTML 文件，可能包含演示页面、工具界面、信息展示页、临时页面或实验性 HTML 片段。每个页面独立运行，互不依赖。

## 目录说明

```
.
├── index.html          # 仓库根目录入口页面（可选，可包含页面列表或导航）
├── pages/              # 主要页面存放目录
│   ├── example-page/   # 单个页面文件夹（包含 HTML 及所需资源）
│   └── ...
├── assets/             # 公共资源目录（图片、样式、脚本等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：存放各独立 HTML 页面及其附属资源。每个页面建议放在独立子目录中，避免文件冲突。
- `assets/`：存放多个页面可能共用的样式、脚本或图片资源。若页面有专用资源，建议放在其自身目录下。

## 页面归档说明

- 每个页面为一个独立的 HTML 文件，可包含内联样式与脚本，或引用 `assets/` 下的公共资源。
- 页面之间无链接依赖，可通过根目录 `index.html` 或直接访问文件路径查看。
- 归档的页面可能来自不同项目、不同阶段，或仅为临时用途。部分页面可能不包含完整样式或功能，仅供查阅或存档。

## 维护说明

- 欢迎提交 Pull Request 添加新页面或优化已有页面。请尽量保持目录结构清晰，并在 `pages/` 下创建独立子目录。
- 若页面不再需要，可提交 PR 删除相应文件。请同时在 PR 描述中说明删除原因。
- 本仓库不承担页面内容的技术支持或更新承诺。页面可能随仓库维护而被移除或修改。
- 请勿在仓库中放置含有敏感信息、违法内容或侵犯他人权益的页面。
